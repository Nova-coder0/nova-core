# NOVA Roadmap

## Phase 0: Trust Foundation

- Publish due diligence.
- Treat V1 as the active token.
- Publish V1 stabilization plan.
- Publish tokenomics draft.
- Publish V2 contract architecture.
- Add Trust Wallet metadata.
- Add official logo.
- Decide V1 verification vs V2 migration.

## Phase 1: V1 Stabilization

- Verify V1 source if original source is available.
- Publish owner and treasury policy.
- Add real liquidity only after tokenomics are public.
- Publish liquidity lock or multisig policy.
- Build V1 monitoring script.
- Build public V1 dashboard.

## Phase 2: V2 Contract Backup

- Implement fixed-supply V2 token.
- Add tests.
- Deploy to BSC testnet.
- Verify source on BscScan testnet.
- Review contract with external tooling.
- Deploy to BSC mainnet only after tokenomics are finalized.

## Phase 3: Transparency Dashboard

- Holder count.
- Supply distribution.
- Liquidity monitor.
- Treasury wallet monitor.
- Large transfer alerts.
- Admin-event monitor for V1.
- V1 health and optional V2 migration status.

## Phase 4: Utility Layer

- NOVA account dashboard.
- NOVA API-credit prototype.
- Telegram/Discord bot gated by NOVA holdings or credits.
- Public usage metrics.

## Phase 5: NOVA Devnet

- Minimal appchain prototype.
- NOVA balance mirror from BSC.
- Cheap action ledger for app credits and reputation.
- Periodic BSC checkpointing.

## Phase 6: Ecosystem

- Grants.
- Third-party integrations.
- Developer SDK.
- Governance only after real protocol usage exists.
